### `git status` vs `git log` > git status: working tree status > git log: history der Änderungen von allen commits
…neue Datei hinzufügen "other.txt" und file.txt verändern dann den status checken ``` > git status On branch master Changes not staged for commit: (use "git add ..." to update what will be committed) (use "git checkout -- ..." to discard changes in working directory) modified: file.txt Untracked files: (use "git add ..." to include in what will be committed) other.txt no changes added to commit (use "git add" and/or "git commit -a") ```
other.txt zum tracking hinzufügen ``` > git add other.txt > git status On branch master Changes to be committed: (use "git reset HEAD ..." to unstage) new file: other.txt Changes not staged for commit: (use "git add ..." to update what will be committed) (use "git checkout -- ..." to discard changes in working directory) modified: file.txt ```
Dateien stagen und commiten zur gleichen Zeit ``` > git commit -am "Added new file other.txt" [master a709c60] Added new file other.txt 2 files changed, 2 insertions(+) create mode 100644 other.txt > git status On branch master nothing to commit, working directory clean ```

Zusammenfassung

Git Branching

Master branch ist immer angelegt ``` > git branch * master ``` Neuen branch erstellen ``` > git branch bug01 > git branch bug01 * master ``` Wir arbeiten immernoch auf master. bug01 soll daher working branch werden ``` > git checkout bug01 Switched to branch 'bug01' > git branch * bug01 master ```
``` > git branch newBr > git checkout newBr ``` Ist das gleiche wie ``` > git checkout -b newBr ```
neue Datei erstellen um den Bug zu fixen ``` > cat > file_on_bug01.txt fix the bug ^C > git status On branch bug01 Untracked files: (use "git add ..." to include in what will be committed) file_on_bug01.txt nothing added to commit but untracked files present (use "git add" to track) ```
neue Datei hinzufügen und commiten ``` > git add file_on_bug01.txt > git status On branch bug01 Changes to be committed: (use "git reset HEAD ..." to unstage) new file: file_on_bug01.txt > git commit -am "Bug fixed" [bug01 776d7ee] Bug fixed 1 file changed, 1 insertion(+) create mode 100644 file_on_bug01.txt > git status On branch bug01 nothing to commit, working directory clean ```
Dateien im branch bug01 listen ``` > git status On branch bug01 nothing to commit, working directory clean > ls -l total 12 -rw-rw-r-- 1 asp asp 12 jun 11 12:08 file_on_bug01.txt -rw-rw-r-- 1 asp asp 17 jun 5 15:55 file.txt -rw-rw-r-- 1 asp asp 21 jun 5 15:54 other.txt ``` auf den master branch wechseln und dort mal listen ``` > git checkout master Switched to branch 'master' > ls -l total 8 -rw-rw-r-- 1 asp asp 17 jun 5 15:55 file.txt -rw-rw-r-- 1 asp asp 21 jun 5 15:54 other.txt ```
Wir mergen jetzt den master mit bug01 ``` > git merge bug01 Updating a709c60..776d7ee Fast-forward file_on_bug01.txt | 1 + 1 file changed, 1 insertion(+) create mode 100644 file_on_bug01.txt ``` und listen unsere Dateien ``` > ls -l total 12 -rw-rw-r-- 1 asp asp 12 jun 11 15:16 file_on_bug01.txt -rw-rw-r-- 1 asp asp 35 jun 11 13:48 file.txt -rw-rw-r-- 1 asp asp 21 jun 5 15:54 other.txt ``` Zuletzt kann man den branch löschen ``` > git branch -d bug01 Deleted branch bug01 (was 776d7ee). ```

Merge Konflikte

Wir erstellen einen neuen branch bug02 und ändern die erste Zeile in file.txt und commiten: ``` > git checkout -b bug02 Switched to a new branch 'bug02' > git branch * bug02 master > > vi file.txt > > git commit -am "Modified file.txt on bug02" [bug02 fa2a1b4] Modified file.txt on bug02 1 file changed, 1 insertion(+) ```
Jetzt ändern wir file.txt im master: ``` > git checkout master Switched to branch 'master' > vi file.txt > git commit -am "Added new line on file.txt at master" [master 3c07391] Added new line on file.txt at master 1 file changed, 1 insertion(+) ```
Wenn jetzt bug02 in den master mergen: ``` > git branch bug02 * master > git merge bug02 Auto-merging file.txt CONFLICT (content): Merge conflict in file.txt Automatic merge failed; fix conflicts and then commit the result. > more file.txt <<<<<<< HEAD New line at 'master' ======= New line here at 'bug02'. >>>>>>> bug02 this is a file . ```
Git status: ``` > git status On branch master You have unmerged paths. (fix conflicts and run "git commit") Unmerged paths: (use "git add ..." to mark resolution) both modified: file.txt no changes added to commit (use "git add" and/or "git commit -a") ```
Editieren von file.txt um den conflict zu resolven: ``` > vi file.txt > more file.txt New line at 'master' New line here at 'bug02'. this is a file > git status On branch master You have unmerged paths. (fix conflicts and run "git commit") Unmerged paths: (use "git add ..." to mark resolution) both modified: file.txt no changes added to commit (use "git add" and/or "git commit -a") ```
Jetzt wieder git add um klarzustellen, das der conflict gelöst ist: ``` > git add file.txt > git status On branch master All conflicts fixed but you are still merging. (use "git commit" to conclude merge) nothing to commit, working directory clean > git commit -am "Merge done" [master cb2da7c] Merge done > git status On branch master nothing to commit, working directory clean ```

Remote repositories

### Remotes - Repositories irgendwo im netz gehostet - Notwendig um kollaborativ zu arbeiten - Hier wird push & pull von remote zu lokalen repositories angewendet
Um ein remote repository hinzuzufügen: ``` > git remote add origin http://asp@smcdevatlas01/stash/scm/test/test.git > git remote origin > git remote -v origin http://asp@smcdevatlas01/stash/scm/test/test.git (fetch) origin http://asp@smcdevatlas01/stash/scm/test/test.git (push) > ```
Wir `pushen` den `master branch` auf `origin` ``` > git push origin master Password for 'http://asp@smcdevatlas01': Counting objects: 17, done. Delta compression using up to 4 threads. Compressing objects: 100% (10/10), done. Writing objects: 100% (17/17), 1.45 KiB | 0 bytes/s, done. Total 17 (delta 3), reused 0 (delta 0) To http://asp@smcdevatlas01/stash/scm/test/test.git * [new branch] master -> master > ```
## Um den aktuellen branch mit mit origin zu verknpüfen > git push --set-upstream origin master
Wir können jetzt unser remote repository in einen anderen Ordner clonen ``` > git clone http://asp@some_gitserver/test.git test Cloning into 'test'... Password for 'http://asp@some_gitserver': remote: Counting objects: 23, done. remote: Compressing objects: 100% (14/14), done. remote: Total 23 (delta 5), reused 0 (delta 0) Unpacking objects: 100% (23/23), done. Checking connectivity... done. ```
…und unsere branches und remote Konfiguration ansehen ``` > git branch -a * master remotes/origin/HEAD -> origin/master remotes/origin/fix01 remotes/origin/master > git remote -v origin http://asp@some_gitserver/test.git (fetch) origin http://asp@some_gitserver/test.git (push) ```
## Was `git clone` macht - Setup des origin remote-repository - Download des master branch
Neue Datei erstellen file02.txt, commit und push auf remote ``` > vi file02.txt > git add file02.txt > git commit -m "New file 02" [master d78e486] New file 02 1 file changed, 0 insertions(+), 0 deletions(-) create mode 100644 file02.txt > git push Password for 'http://asp@some_gitserver': Counting objects: 5, done. Delta compression using up to 4 threads. Compressing objects: 100% (2/2), done. Writing objects: 100% (3/3), 337 bytes | 0 bytes/s, done. Total 3 (delta 0), reused 0 (delta 0) To http://asp@some_gitserver/test.git 720262a..d78e486 master -> master ```
Zurück in den vorherigen Ordner wechseln, dann auch eine neue Datei erstellen und pushen: ``` > git push origin master Password for 'http://asp@some_gitserver': To http://asp@some_gitserver/test.git ! [rejected] master -> master (fetch first) error: failed to push some refs to 'http://asp@some_gitserver/test.git' hint: Updates were rejected because the remote contains work that you do hint: not have locally. This is usually caused by another repository pushing hint: to the same ref. You may want to first integrate the remote changes hint: (e.g., 'git pull ...') before pushing again. hint: See the 'Note about fast-forwards' in 'git push --help' for details. ```
Die remote hat Änderungen, die wir erst herunterladen und mergen müssen (mit unserem lokalen repository) ``` > git pull origin master Password for 'http://asp@some_gitserver': From http://some_gitserver/test * branch master -> FETCH_HEAD Merge made by the 'recursive' strategy. file02.txt | 0 1 file changed, 0 insertions(+), 0 deletions(-) create mode 100644 file02.txt ```
Ohne Konflikte können wir jetzt pushen: ``` > git push origin master Password for 'http://asp@some_gitserver': Counting objects: 8, done. Delta compression using up to 4 threads. Compressing objects: 100% (4/4), done. Writing objects: 100% (5/5), 556 bytes | 0 bytes/s, done. SASS, SCSS

Sass makes CSS fun again. Sass is an extension of CSS, adding nested rules, variables, mixins, selector inheritance, and more. It's translated to well-formatted, standard CSS using the command line tool or a web-framework plugin.
## Sass Variablen: ``` $font-stack: Helvetica, sans-serif; $primary-color: #333; body { font: 100% $font-stack; color: $primary-color; } ``` Kompiliert zu CSS: ``` body { font: 100% Helvetica, sans-serif; color: #333; } ```
## Sass Nesting: ``` nav { ul { margin: 0; padding: 0; list-style: none; } } ``` ``` nav ul { margin: 0; padding: 0; list-style: none; } ```
## Sass Mixins: ``` @mixin border-radius($radius) { -webkit-border-radius: $radius; -moz-border-radius: $radius; -ms-border-radius: $radius; border-radius: $radius; } .box { @include border-radius(10px); } ``` ``` .box { -webkit-border-radius: 10px; -moz-border-radius: 10px; -ms-border-radius: 10px; border-radius: 10px; } ```
## Sass Extend/Inheritance: ``` .message { border: 1px solid #ccc; padding: 10px; color: #333; } .success { @extend .message; border-color: green; } ``` ``` .message, .success, .error, .warning { border: 1px solid #cccccc; padding: 10px; color: #333; } .success { border-color: green; } ```
## Sass Operators: ``` .container { width: 100%; } article[role="main"] { float: left; width: 600px / 960px * 100%; } ``` ``` .container { width: 100%; } article[role="main"] { float: left; width: 62.5%; } ```
